# 기존 코드 확인

새 함수, 클래스, 모듈을 만들기 전에 기존 코드에 같은 기능이 있는지 확인한다.

## 확인 순서

### 1. LSP로 심볼 검색
- 만들려는 함수명의 핵심 동사+명사로 검색
- 같은 도메인 폴더 내 export된 함수 시그니처 확인
- shared/ 폴더 확인

### 2. 발견 시

| 상황 | 행동 |
|------|------|
| 동일 기능이 있음 | 기존 것을 사용. 사용자에게 보고. |
| 유사하지만 다름 | 차이 설명 후 사용자에게 질문. |
| 없음 | 새로 만들어도 된다. 진행. |

### 3. 없음이 확실할 때만 생성
확인 결과를 한 줄로 보고한 뒤 작업 진행.

## 주의
- 전체 코드를 읽지 마라. export 시그니처와 파일명만으로 충분.
- LSP 우선. 안 되면 grep/ripgrep.
- 30초 이상 걸리면 방법이 잘못된 것.
