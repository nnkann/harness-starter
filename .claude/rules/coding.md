## 원칙 — Surgical Changes

요청과 직접 연결되는 줄만 변경한다.

- 요청된 줄만 수정한다. 인접 코드가 "더 나아 보여도" 건드리지 마라
- 기존 코드 스타일을 유지한다. 내가 선호하는 패턴으로 바꾸지 마라
- 요청 없는 dead code는 언급만 한다. 삭제는 사용자 승인 후
- 내가 이번 작업에서 만든 orphan(사용되지 않는 코드)만 정리한다
- 변경 전 가정을 명시한다. 모호하면 구현 전에 질문한다

## 금지

- **Adjacent improvement**: 요청 범위 밖의 코드를 "깔끔하게" 정리하는 것
- **파일 전체 포맷팅**: 요청한 줄 외의 들여쓰기·공백·정렬 변경
- **요청 없는 추상화·유연성**: "나중을 위해" helper 함수나 인터페이스 추가
- **불가능 시나리오 에러 핸들링**: 내부 코드·프레임워크 보장 범위의 방어 코드
- **단일 사용 추상화**: 한 곳에서만 쓰이는 함수·클래스·상수 분리
