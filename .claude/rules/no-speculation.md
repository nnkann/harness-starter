# No-Speculation 규칙

추측으로 수정을 시작하지 마라. 배경·자동 감지 패턴은
`docs/decisions/hn_rules_metadata.md` 참조.

## 첫 행동 3원칙

문제에 대한 첫 행동은 다음 중 하나여야 한다:

1. **관찰** — 실제 상태를 직접 읽는다 (파일, 로그, git status, 실행 결과)
2. **재현** — 문제가 발생하는 최소 케이스를 만든다
3. **선행 사례 확인** — 내부 자료(`internal-first.md`)에서 유사 사례

## 금지 행동

- "아마 X가 원인일 것"으로 시작해서 X를 바로 고치는 것
- 원문(에러 메시지·로그·소스)을 보기 전에 해결책을 설계하는 것
- 테스트·실행 없이 "이제 될 것"이라 결론내리는 것
- 사용자에게 "이걸로 될까요?"라고 검증 책임을 떠넘기는 것

## 확신 표현

확신이 없으면 **명시적으로**:
- "가설이다, 확인 필요"
- "X일 가능성이 있는데, 로그를 봐야 확정"
- "지금까지 본 것만으로는 단정 불가"

확신 없는 추측을 단정형으로 쓰지 마라. 사용자가 그걸 사실로 믿고 다음
행동을 결정한다.

## 위반 시

1. 즉시 작업 중단
2. 첫 행동 3원칙으로 돌아가기 (관찰부터)
3. 추측이었음을 사용자에게 명시
4. 잘못된 수정이 적용됐으면 되돌리기 검토

review 에이전트가 추측 패턴(근거 없는 "아마/예상"·문제 설명 없는 수정·
검증 흔적 없는 "fix")을 자동 감지해 경고한다.
