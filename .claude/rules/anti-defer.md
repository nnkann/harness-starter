# Anti-Defer 규칙 — 미루기 차단

defends: P7

자율 신뢰 영역에서 Claude가 작업을 미루는 회피 패턴 차단. 자기증명
사례(2026-05-02): 본 세션 우선순위 5 "데이터 누적 필요"라 미뤘으나 실제로는
같은 세션에 review 호출 3건+ 있어 즉시 측정 가능했음 — 회피 사유로 사용.

## 핵심 원칙

**사용자가 명시 처리 지시를 내리면 즉시 처리한다.** wave 정의·"측정 후"·
"다음 세션" 같은 시스템 메커니즘을 미루기 도구로 사용하지 않는다.

## 두 사안 분리

| 사안 | 정상/회피 |
|------|----------|
| 핫스팟 발견 → 별 wave WIP 신설 후 처리 | ✅ 정상 흐름 |
| 핫스팟 발견 → "다음 세션", "데이터 누적 후" | ❌ 미루기 |
| 본 wave commit 후 별건 발견 → 별 wave 신설해 즉시 처리 | ✅ 정상 |
| 본 wave commit 후 같은 wave 무단 확장 | ❌ 봉인 위반 (별 룰: completed 봉인) |

별 wave 분리 ≠ 미루기. **별 wave 신설 + 즉시 처리**가 정합.

## 미루기 회피 사유 블랙리스트

다음 사유로 작업을 미룰 때 **사용자 명시 승인 없이 단독 사용 금지**:

- "측정 후 처리"·"5+ commit 누적 후"
- "다음 세션에서"·"다음 컨텍스트로 인계"
- "데이터 누적 필요"
- "별 wave로 분리·후속 처리" (별 wave 분리 자체는 OK, 처리 시점이 "후속"이면 미루기)
- "운용 검증 필요"·"실측 후"

**예외 — 사용자 명시 승인 시 사용 가능**:
- 사용자가 "다음 세션에 해" 같이 명시 지시
- 다운스트림 환경 등 starter에서 측정 불가능한 경우 (구체 사유 명시)
- 사용자가 wave 책임 정의에 동의한 경우

**예외 — 객관 사유 (적용 불가)**:
- 다운스트림 환경 의존 (예: starter 단독 측정 불가)
- 외부 시스템 응답 대기 (CI 결과·외부 API 등)
- 사용자 입력 필요 (의사결정 권한 사용자에게 있음)

## 자기 검토 트리거

작업 중 다음 표현이 응답에 떠오르면 **즉시 멈추고 정직하게 진단**:

- "측정 후 별 wave에서..."
- "다음 컨텍스트가 처리..."
- "데이터 누적 필요라 보류..."
- "별 wave WIP 신설하고 다음 세션..."

질문:
1. 사용자가 명시 처리 지시했는가? → 그러면 즉시 처리
2. 본 세션에 측정·실행 가능한가? → 그러면 즉시 처리
3. 진짜 객관 사유 (다운스트림·외부 대기·사용자 입력)인가? → 사유 구체 명시
4. 위 셋 다 아니면 → 미루기 회피. 즉시 처리

## 위반 감지

- **review 에이전트**: staged WIP 본문에 미루기 사유 블랙리스트 표현이
  있고, 같은 세션·환경에서 처리 가능한 경우 [주의] 보고
- **사용자**: "왜 안 하지?" 류 지적 시 Claude는 본 룰 자가 점검 후 응답
