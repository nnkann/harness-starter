# Memory 활용 규칙

하네스의 핵심 철학 "실수를 코드화"의 세션 간 확장.
코드/규칙으로 강제할 수 없는 학습을 메모리에 저장한다.

## 메모리 경로

프로젝트별: `.claude/memory/` 디렉토리
인덱스: `.claude/memory/MEMORY.md`

## 우선 저장 대상

| 우선순위 | 타입 | 저장 대상 | 예시 |
|----------|------|----------|------|
| 1 | feedback | 사용자가 수정한 접근법 | "이 프로젝트에서는 단일 PR로 묶어라" |
| 2 | project | 현재 프로젝트 맥락, 마감 | "4/15까지 인증 완료 필요" |
| 3 | user | 사용자 역할, 전문 분야 | "백엔드 경험 풍부, 프론트 초심자" |
| 4 | reference | 외부 시스템 포인터 | "버그 트래킹은 Linear INGEST 프로젝트" |

## 저장하지 않는 것

- 코드에서 읽을 수 있는 것 (구조, 패턴, 아키텍처)
- git log로 알 수 있는 것 (변경 이력, 누가 뭘 바꿨는지)
- CLAUDE.md나 rules/에 이미 있는 것
- 현재 세션에서만 유효한 임시 정보

## 행동 규칙

- 사용자가 접근법을 수정하면 → feedback 메모리 저장 검토
- 사용자가 "기억해" 요청하면 → 즉시 저장
- 메모리를 참조할 때 → 현재 상태와 대조 후 사용 (오래된 메모리 주의)
- 메모리와 현재 코드가 충돌하면 → 현재 코드를 신뢰, 메모리 업데이트

## 하네스와의 관계

| 지속성 | 도구 | 용도 |
|--------|------|------|
| 세션 내 | docs/WIP/ + TODO | 현재 작업 추적 |
| 세션 간 (코드화 가능) | rules/, CLAUDE.md | 규칙으로 강제 |
| 세션 간 (코드화 불가) | **memory** | 맥락, 피드백, 선호도 |
| 영구 | git history | 변경 이력 |
