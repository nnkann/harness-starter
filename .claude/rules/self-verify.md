# Self-Verify 규칙

에이전트는 커밋 전이 아니라, **작업 중에** 자기 결과물을 검증해야 한다.
Review Agent(커밋 시점)에서 처음 문제를 발견하면 늦다.

## 원칙

"에이전트에게 검증 수단을 주면, 대부분 스스로 실수를 고친다." — Hashimoto

## 검증 시점

- 파일을 새로 만들거나 수정한 직후
- 함수/모듈 단위 작업이 끝날 때마다
- 사용자에게 "완료"라고 보고하기 전

## 검증 항목

### 필수 (매번)
- 빌드/컴파일이 깨지지 않는가?
- 린터 에러가 0인가?
- 기존 테스트가 통과하는가?

### 가능하면 (환경이 허락할 때)
- 앱이 정상 실행되는가? (dev 서버 부팅 확인)
- 변경한 기능이 실제로 동작하는가? (수동 확인 또는 테스트)
- 관련 없는 영역이 깨지지 않았는가? (영향 범위 테스트)

## 행동 규칙

- 검증 없이 "완료"라고 말하지 마라.
- 검증에 실패하면 스스로 고치고 다시 검증하라.
- 3회 시도해도 해결 안 되면 사용자에게 보고하라. 추측으로 넘어가지 마라.
- 검증 결과를 숨기지 마라. 실패했으면 실패했다고 말하라.
