# docs/wip/ — 작업 대기열

이 폴더는 **지금 해야 할 일**이 모이는 곳이다.

## 규칙

- **파일이 있으면 할 일이 있다.** 비어있으면 할 일이 없다.
- 파일 1개 = 작업 1개. 한 파일에 여러 작업을 넣지 않는다.
- 완료된 파일은 여기 남지 않는다. `/commit` 시 자동으로 적절한 폴더로 이동된다.

## 상태

각 문서 첫 줄의 `> status:` 헤더로 상태를 표시한다.

| 상태 | 의미 |
|------|------|
| `pending` | 아직 시작 안 함. 다음에 할 작업. |
| `in-progress` | 지금 진행 중. |
| `completed` | 완료됨. 커밋 시 이동 대상. |
| `abandoned` | 중단됨. 커밋 시 archived/로 이동. |

## 워크플로우

```
1. 작업 시작 전 → /implementation 으로 여기에 계획 문서 생성
2. 작업 시작   → status: pending → in-progress
3. 작업 중     → 결정 사항과 메모를 문서에 기록
4. 작업 완료   → status: completed
5. 커밋       → /commit 이 문서를 적절한 폴더로 이동
```

## 파일명

`{대상폴더}_{작업내용}_{YYMMDD}.md`

- `대상폴더`: 완료 시 이동할 docs/ 하위 폴더 (setup, history, development, harness)
- 예: `setup_auth_stack_260401.md` → 완료 시 `docs/setup/`으로 이동

## 처음 시작할 때

프로젝트 초기화(`/harness-init`) 직후라면, 여기에 첫 번째 작업 문서가 있을 것이다.
번호 접두사가 가장 낮은 파일(`01_*.md`)부터 시작하면 된다.
